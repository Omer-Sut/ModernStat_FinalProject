Preprocessing summary

- Hovot == 2 -> y_debt = 1, debt.
- Hovot == 1 -> y_debt = 0, no debt.

Feature engineering summary
---------------------------
Profile variables:
- Nominal/categorical fields were one-hot encoded: district, gender, family status,
  population group, religion/language, living arrangement, employment status, profession.
- Ordered/binned fields were mapped to meaningful numeric values: household size, age,
  years of education, household income, work income.
- Open-ended top bins use an extrapolated representative value and an additional top-bin indicator.
- Example: age bin 20-24 is mapped to 23; 75+ is mapped to 80 and has top-bin flag.
- Income codes 98/99 are treated as unknown/refused flags, not as large numeric incomes.

Quiz variables:
- Q2 balance tracking score order: 2 > 3 > 4 > 5 > 1.
- Q5 banking event frequency: 1 best, 5 worst.
- Q7 credit cards: 1->0, 2->1, 3->2, 4->3, 5->4.5 plus top-bin flag.
- Q20 financial knowledge: correct answer for all subquestions is yes; score yes > IDK > no.
- Q21 interest calculation: 1020 highest; 1000-1500 good; IDK code 2 middle; >1500 low; <1000 lowest.
- Q22 score order: 1 > 2 > 3 > 5 > 4.
- Q23 score order: 2 > 3 > 1 > 4.
- Q24: higher value means more financial-information search.
- Q25: positive and negative attitude statements scored according to direction; code 5 is zero plus not-relevant flag.
- Q26: topics 1-6 are equal interest indicators; topic 7 marks no need to expand knowledge.
- Q27: non-ordinal source-interest indicators.
- Q29: 1 highest mortgage knowledge, 5 lowest.

Missing/IDK handling:
- Missing, IDK, not-applicable, and not-relevant signals are kept as explicit indicators where relevant.
- Remaining NaNs in engineered numeric features are neutral-filled using labeled-row medians so algorithms can run.
- A neutral-fill audit exists at data/supervised/neutral_fill_audit.csv.
- Before neutral fill: 55 feature columns had NaNs; 13,659 missing feature values in labeled rows; 864 in unlabeled rows.

Lasso path implementation
-------------------------
The lasso path is for classification, not regression.
Implementation in notebook:
- L1-penalized logistic regression.
- Implemented manually with NumPy using warm-started proximal-gradient/FISTA.
- This avoids using sklearn's standard regression lasso_path, which is not appropriate for classification.
- Features are standardized before fitting the path.
- Coefficients shown in the path are standardized coefficients.
- Constant features are dropped before standardization because their standard deviation is 0.

- Lambda grid size: 35.
- lambda_max: 0.1108067941802435.
- lambda_min: 0.0044322717672097.
- Active coefficients at strongest lambda: 0.
- Active coefficients at weakest lambda: 177.

Meaning of lasso quantities
---------------------------
entry_step:
- First lambda-grid index where the feature coefficient becomes nonzero.
- Smaller entry_step means the feature enters earlier, under stronger regularization.
- Earlier entry generally indicates a stronger/more stable predictive signal in the lasso path.

final_coef:
- Coefficient value at the last point of the path, where lambda is smallest.
- It is not used as a feature value in later models.
- It is only an interpretation of the L1-logistic model at the weakest regularization point tested.
- Later models should use selected feature columns from X and learn their own parameters on the training set.

Sign interpretation:
- y_debt = 1 means debt.
- Positive coefficient: higher feature value increases predicted debt odds in the lasso model.
- Negative coefficient: higher feature value decreases predicted debt odds in the lasso model.
- Because features are standardized, coefficient magnitudes are comparable across features.

Important caution for report wording
------------------------------------
Do not write that selected variables causally affect debt.
Correct wording:
"These variables are predictive of the debt label in the L1-regularized logistic model."

Many top features are attitude scores, knowledge scores, missingness indicators, IDK indicators, or one-hot categories.
For missingness indicators, write carefully:
"Several missingness indicators entered the path, suggesting that survey response patterns or skip patterns contain
predictive information about the debt label."

PCA implementation and outputs
------------------------------
PCA was added to code/SupervisedLearning.ipynb in the same supervised notebook.

Implementation:
- Input is data/supervised/supervised_labeled.csv.
- X excludes SerialNumber, y_debt, and sample_weight.
- X is standardized before PCA because engineered features mix numeric bins, semantic scores, binary indicators,
  one-hot indicators, and missingness indicators.
- Constant features are dropped before standardization.
- PCA is computed with NumPy SVD of standardized X, no sklearn dependency.
- Leading 4 principal components are computed as required by the assignment.
- Component scores are alpha_i = X_standardized v_i.
- PR-AUC is computed manually from alpha_i and y_debt.
- Because PCA signs are arbitrary, both alpha_i and -alpha_i PR-AUC are reported; best_orientation is used only
  for relevance interpretation.

- Constant features dropped: 23.

- PCA signs are arbitrary, so comparing alpha_i and -alpha_i is the correct way to orient each component for PR-AUC.

Part 4 supervised model training and evaluation
-----------------------------------------------
Part 4 was added to code/SupervisedLearning.ipynb in the same supervised notebook.

Implementation:
- Input is data/supervised/supervised_labeled.csv.
- y_debt is the label; SerialNumber and sample_weight are metadata, not features.
- Main F1 scores are unweighted positive-class F1 for y_debt = 1.
- Deterministic stratified split with seed 42.
- Train set: 768 rows, with 615 no-debt and 153 debt.
- Test set: 385 rows, with 308 no-debt and 77 debt.
- 5-fold stratified CV is done inside the training set only.
- Standardization is fit inside each CV fold using only that fold's training rows; final standardization is fit on

- Thresholds 0.20 through 0.60 by 0.05 are tuned by CV because the positive class is about 20%.
- Models:
  1. L2 logistic regression.
  2. KNN with Euclidean distance.
  3. RBF-kernel logistic regression.
- Feature sets:
  1. full_features: all 444 engineered features before train-constant dropping.
  2. lasso_top30: top 30 features selected by a training-only L1-logistic lasso path.
- The lasso_top30 selector uses the outer training labels only, never the held-out test labels.

Part 5 learner diversity and majority vote
------------------------------------------
Part 5 was added to code/SupervisedLearning.ipynb.

Implementation:
- Uses the three lasso_top15 models from the feature-count experiment, per the user's decision after inspecting
  held-out test F1.
- Models used:
  1. logistic_l2, lasso_top15, l2=0.001, threshold=0.30.
  2. KNN, lasso_top15, Euclidean distance, k=35, threshold=0.20.
  3. kernel_logistic_rbf, lasso_top15, gamma_multiplier=2.0, l2=0.01, threshold=0.25.
- Builds the assignment's training error matrix E with shape 768 x 3.
- Centers and L2-normalizes each error column to get E_tilde.
- Computes and saves E_tilde^T E_tilde.
- Computes a majority vote meta learner where prediction is 1 if at least 2 of the 3 learners predict 1.

- KNN vs kernel_logistic_rbf: 0.7571.

Part 5 F1 results:
- logistic_l2 top15: train F1=0.621, test F1=0.623.
- KNN top15: train F1=0.573, test F1=0.536.
- kernel_logistic_rbf top15: train F1=0.580, test F1=0.600.
- majority vote: train F1=0.591, test F1=0.610.

Part 5 interpretation notes:
- The learners make positively correlated errors; the largest off-diagonal entry is KNN vs kernel logistic, about 0.757.
- Majority voting improves over KNN and kernel logistic individually, but it does not improve over the best individual
  top15 learner, logistic_l2 with test F1 about 0.623.
- A better combination to suggest in the report is stacking, for example a meta-logistic learner trained on out-of-fold
  base-model scores, because it can learn unequal weights instead of forcing a 2-of-3 vote.
